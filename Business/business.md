# _Business_

## _Targets_

ForgeBot targets Discord users aged 13 and above, which is the minimum legal age required to use Discord.

The bot is designed for a wide range of users who want to create and manage well-organized and visually appealing servers for different purposes, such as gaming communities, social groups, or even professional environments.

Creating a new Discord server can be **complex and time-consuming**. It requires proper **configuration**, **moderation tools**, and **security systems**. In addition, a successful server often needs a certain **level of creativity**, customization, and **community engagement**. ForgeBot aims to simplify this process.

## _Goals_

### **Acquisition Goals**:

To attract users to our service, we aim to :

- **Promote** ForgeBot across a wide variety of Discord servers.
- **Increase** visibility within gaming, community, and professional servers.


### **Retention Goals** :

To encourage users to **keep** ForgeBot installed on their servers, we aim to :

- Provide engaging **features** that encourage regular interaction.
- Offer entertainment systems such as an **RPG-based activity system** to keep members active.

### **Automation Goals** :

ForgeBot aims to **reduce** the workload of moderators and administrators by **automating** essential server management tasks.

The bot **helps** create a fully functional, organized, and secure Discord server with **minimal manual setup**.

### **Communication Goals**  : 

ForgeBot enhances communication and community engagement by:

- Encouraging **interactions** between members.
- Making servers more **dynamic** through entertainment features based on an RPG system.

## _Marketing effectiveness_ :

ForgeBot will rely on several marketing channels to reach potential users:

- Discord bot listing platforms.
- Community promotion in gaming servers.
- Social media visibility.
- Word-of-mouth between server administrators.

These channels aim to increase the visibility of ForgeBot and encourage server owners to install the bot.

## _Hypothesis_ :

For each scenario, we only consider servers that subscribe to the premium plan within 6 months.

The premium subscription price is €3.50 per month, which corresponds to 21€ for 6 months.

The revenue is estimated using the following model:

        Revenue = Servers installed
                × Retention rate
                × Average server size
                × Active user rate
                × Premium conversion rate
                × Price (6 months)



### **Pessimistic scenario**

    Servers/week = 100
    Retention = 20%

    Average server size = 50
    Active user rate = 5%

    Commands per user/day = 3

    Premium conversion = 1%


So :

    Servers after 6 months = 100 × 24 = 2400

    Active servers = 2400 × 0.2 = 480

    Active users = 480 × 50 × 0.05 = 1200 users

    Commands / day = 1200 × 3 = 3600

    Premium servers = 480 × 1% = 4.8 ≈ 5

    Revenue = 5 × 21€ = 105 €



### **Realistic scenario**

    Servers/week = 400
    Retention = 40%

    Average server size = 120
    Active user rate = 10%

    Commands per user/day = 6

    Premium conversion = 2%


So :

    Servers = 400 × 24 = 9600

    Active servers = 9600 × 0.4 = 3840

    Active user = 3840 × 120 × 0.1 = 46 080 users

    Commands / day = 46 080 × 6 = 276 480

    Premium Servers = 3840 × 2% ≈ 77

    Revenue = 77 × 21€ = 1 617 €

    
### **Optimistic scenario**

    Servers/week = 1000
    Retention = 60%

    Average server size = 250
    Active user rate = 15%

    Commands per user/day = 10

    Premium conversion = 4%


So :

    Servers = 1000 × 24 = 24 000

    Active servers = 24 000 × 0.6 = 14 400

    Active user = 14 400 × 250 × 0.15 = 540 000 users

    Commands / day = 540 000 x 10 = 5 400 000
    
    Premium Servers = 14 400 × 4% ≈ 576

    Revenue = 576 × 21€ = 12 096 €

### **Cost estimation**

Development :

        1 Developer Cost = 96h x 12€ (minimum wage) = 1152

        Development team = 2 persons

        Total Development cost = 2 x 1152 = 2304 €

Infrastructure :

        Estimated VPS / cloud cost = 20–80€ per month

        Chosen estimate = 40€/month

        Infrastructure cost = 40 × 6 = 240 €


Marketing :

        Estimated Budget = 100€/month

        Marketing cost = 100 x 6 = 600€

Maintenance :

        Estimated Cost = 4€/month

        Maintenance cost = 4 x 6 = 24€


Total Cost :

        2304 + 240 + 600 + 24 = 3168 €



Conclusion : 

The initial cost of the project for the first six months is estimated at 3 168 €.

We can now calculate the Return on Investment (ROI) to evaluate the financial viability of the bot.

### **Return of Investments (ROI)**

ROI = (Revenue - Cost) / Cost

Pessimitic :

        ROI = (105 - 3168) / 3168
            = -0.966
            = - 96.6 %

Realistic :

        ROI = (1617 - 3168) / 3168
            = -0.489
            = - 49 %
        
Optimistic :

        ROI = (12 096 - 3168) / 3168
            = 2.818
            = 282 %


Results :

With our three different scenarios, we can observe that the realistic hypothesis results in relatively high costs and will not be profitable within the first six months. However, the optimistic scenario shows that if the bot experiences strong growth, the return on investment could become very attractive.

We will now estimate, based on the realistic scenario, when the project could become profitable.

### **When will the project become profitable?**

Revenue per month = 1 617 / 6 = 269.5 €
Cost = 3 168 € 

Number of month to be profitable = Cost / Revenue per month
                                 = 3168 / 269.5
                                 = 11.75 months

Conclusion :

        To be profitable we need 1 year based on the realistic scenario.
